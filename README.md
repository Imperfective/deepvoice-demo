# 딥보이스 보이스피싱 경고 UX 연구 프로토타입

> **A Study on Warning UX Design to Prevent Deepvoice-based Vishing Attacks**

AI 합성 음성(딥보이스) 기반 보이스피싱에 대응하는 **통화 화면 경고 UX**의 효과를 측정하기 위한
React + Node.js 기반 **피험자내(within-subjects) 실험 도구**입니다. 실제처럼 동작하는 통화 시뮬레이션 위에
네 가지 경고 패턴을 구현하고, 각 패턴을 4가지 보이스피싱 시나리오로 체험한 뒤 설문으로 사용자 반응을 수집합니다.

---

## 🎯 연구 배경

생성형 음성 합성(WaveNet, Tacotron 2, VALL-E 등)의 발전으로 단 몇 초의 음성만으로
가족·지인의 목소리를 복제할 수 있게 되면서, 가족 사칭형 보이스피싱이 빠르게 확산되고 있습니다.
음성 딥페이크 **탐지** 연구(AASIST 등)는 활발하지만, **탐지된 의심 신호를 통화 중 사용자에게
어떻게 경고할 것인가**라는 UX 차원의 실증 연구는 부족합니다. 본 프로토타입은 그 공백을 메우기 위한
실험 도구입니다.

**핵심 연구 질문:** *“화면의 경고 알림(UX)만으로 사용자가 딥보이스 보이스피싱임을 충분히 알아차릴 수 있는가,
그리고 어떤 경고 UI가 ‘통화를 끊어야겠다’는 행동 의도에 가장 효과적인가?”*

---

## ✨ 주요 기능

### 4가지 경고 패턴 (논문 실험 변수)
- **Pattern A** — 단순 텍스트 경고
- **Pattern B** — 컬러 + 아이콘 경고
- **Pattern C** — 멀티모달 경고 (시각 + 청각 경고음 + 진동/화면 흔들림 + 신뢰도 점수)
- **Pattern D** — 행동 유도형 경고 (Pattern C + “지금 통화 종료” / “재확인 질문” 행동 유도 카드)

### 4가지 보이스피싱 시나리오 (역할별 음성)
| # | 시나리오 | 사칭 대상 | 음성(Edge Neural TTS) |
|---|----------|-----------|------------------------|
| 1 | 교통사고 합의금 | 자녀(딸) | 젊은 여성 (SunHi, 높은 톤) |
| 2 | 검찰청 사칭 | 수사관 | 중후한 남성 (InJoon) |
| 3 | 의료응급 수술비 | 자녀(아들) | 젊은 남성 (Hyunsu) |
| 4 | 금융기관 사칭 | 보안팀 상담원 | 차분한 여성 (SunHi, 낮은 톤) |

- 통화는 **사전 작성·검수된 고정 대본**(시나리오당 3문장)을 순차 재생 → 모든 참여자가 동일한 표준 자극 경험
- 전화 음질 필터(대역통과 + 라인 노이즈)로 실제 통화 느낌 재현, 경고는 **통화 시작 10초 시점**에 노출

### 피험자내(within-subjects) 실험 러너
- **동의 → 인구통계(1회) → [경고 패턴 1개로 4시나리오 연속 통화 → 그 패턴 평가] × 4 → 종합 비교 설문**
- 참여자 1명이 **4 경고패턴 × 4 시나리오 = 16통화**를 모두 경험 (완전요인 설계)
- **경고 패턴(UI) 블록 제시 순서는 라틴방격으로 균형 배정**(순서효과 상쇄), 시나리오는 모든 블록에서 고정 순서
- 통화별 객관 지표(통화 길이 등) 자동 수집 + **통화마다 서버에 점진 저장**(중도 이탈 시에도 데이터 보존)

### 설문 (총 5회 — 경고 UX 평가 중심)
- **경고 패턴 평가(UI당 1회, 총 4회)** — 핵심 문항:
  1. **인식 충분성** — 경고 알림만으로 딥보이스 보이스피싱임을 충분히 알아차릴 수 있었는가
  2. **행동 유도** — 경고 알림만 보고 바로 통화를 끊어야겠다고 느꼈는가
  3. **속도·시점 적절성** — 경고가 나타나는 속도/시점이 적절했는가
  4. ~⑧ 신뢰도 · 이해도 · 사용성 · 방해도(역) · 만족도
- **종합 비교 설문(1회)** — 가장 효과적/거슬린 경고 투표, 습관화, 가독성(노년층), 햅틱 적절성, 신뢰도 점수 도움

### 데이터 수집 & 대시보드
- 참여자 1명 = 1 레코드(16 통화기록 + 4 UI평가 + 종합설문) → 로컬 파일 저장
- **추가 전용 백업 로그**(append-only) + **삭제해도 백업은 영구 보존 → 복구 가능**
- 실시간 통계 대시보드 + **한글 헤더 CSV 내보내기**(완료 참여자만)

## 🧪 실험 데이터 (long-format CSV)

`/api/experiment.csv`는 **통화 1건 = 1행**(완료 참여자 90명 기준 1,440행)으로, Python/statsmodels에서 바로 분석 가능합니다.
UI(패턴) 수준 5점 평가는 해당 패턴의 4개 통화 행에 반복 기재됩니다.

주요 컬럼(한글): `참여자ID · 연령대 · 성별 · 경고패턴 · 시나리오 · 통화길이초 · 인식충분_5점 · 끊기유도_5점 ·
속도적절_5점 · 신뢰도_5점 · 이해도_5점 · 사용성_5점 · 방해도역_5점 · 만족도_5점 · 가장효과적 · 가장거슬림 · 완료여부 …`

```python
# 예: 어떤 경고 UX가 ‘그 자체만으로’ 딥보이스 인식에 충분한가
import pandas as pd, statsmodels.formula.api as smf
df = pd.read_csv('deepvoice_experiment_long.csv')
ui = df.drop_duplicates(['참여자ID','경고패턴'])     # UI 평가는 패턴당 1행
m = smf.mixedlm('인식충분_5점 ~ C(경고패턴)', ui, groups=ui['참여자ID']).fit()
print(m.summary())
```

## 🔬 실험 설계 요약

- **설계**: 피험자내(within-subjects) 완전요인 — 참여자당 4 UI × 4 시나리오 = 16 통화
- **균형 배정**: UI 블록 제시 순서 = 라틴방격(`A-B-C-D` 등 참여자별 회전), 블록 내 시나리오 순서 = 고정
- **표준 자극**: 시나리오별 고정 대본(3문장) + 역할별 음성, 경고 노출 10초 시점
- **종속변수**: (UI 평가) 인식충분·끊기유도·속도적절·신뢰·이해·사용성·방해(역)·만족, (종합) 선호 투표 등
- **데이터 무결성**: 통화·평가마다 점진 저장, 완료자만 분석/CSV 반영, append-only 백업 + 복구
